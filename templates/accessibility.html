Accessibility Statement

Ghost Protocol — Toronto 2047 · Last updated: June 2026

Ghost Protocol is a multi-agent cyberpunk heist RPG built for the Microsoft Agents League Hackathon. We are committed to making the game accessible to players who use screen readers, keyboard-only navigation, high contrast displays, who are deaf or hard of hearing, who use voice control or switch access, or who have vestibular or cognitive accessibility needs.

We target WCAG 2.1 Level AA compliance. This statement describes what is implemented, what is still in progress, and how to report issues.

Accessibility Features Included

Screen Reader Support Implemented

  • All agent responses announced via aria-live="polite" on the output region.
  • Dice rolls and alert-level changes announced via aria-live="assertive".
  • Every agent card has an aria-label stating who is speaking and a preview of the response (e.g. "Cipher, Hacker, says: Looping the camera feed…").
  • Health bars use role="meter" with aria-valuenow, aria-valuemin, and aria-valuemax.
  • Mission log has role="log" and a polite live region.
  • Phase progress uses role="list" with per-item aria-label describing complete / in-progress / upcoming state.
  • Vex encounter modal has role="dialog", aria-modal="true", and a screen-reader-only description of all three choices.
  • Decorative ASCII art is marked aria-hidden="true" with a visible text alternative.
  • A dedicated off-screen status region announces processing state, navigation changes, and accessibility toggle confirmations.

Tested with: NVDA (Windows), VoiceOver (macOS), and JAWS (Windows).

Keyboard Navigation Implemented

  • Logical tab order: input field → Send button → New Session → accessibility controls.
  • All interactive elements — buttons, links, modal choices — are reachable by Tab alone.
  • Skip-to-content link appears on first Tab press, jumping directly to the input field.
  • Escape clears the input field.
  • Up / Down arrow keys scroll through the command history (last 50 commands).
  • Ctrl+/ (or ⌘+/) opens the keyboard shortcuts reference panel.
  • Vex encounter modal and keyboard shortcuts panel both have full focus traps — Tab cycles within the dialog until it is closed.
  • Vex modal auto-focuses the first choice button on open; focus returns to the triggering element on close.

All keyboard shortcuts are listed in the in-game shortcuts panel (Ctrl+/).

Visual Accessibility Implemented

  • High contrast mode: Replaces the neon-on-dark theme with white text on black background and yellow highlights. Activated by the HC button in the header. Preference is saved to localStorage.
  • Font size toggle: Three sizes — Normal (14 px base), Large (16 px, ~1.14×), Extra Large (20 px, ~1.43×). Activated by the A button in the header.
  • Reduce motion: Disables all CSS animations and transitions, including glow pulses, blinking health indicators, and the scanline overlay. Activated by the ~ button. Also respects the OS-level prefers-reduced-motion media query automatically.
  • Color-independent agent identification: Every agent card shows a unique icon alongside its color: ◈ Ghost, ⚔ Wraith, ⌘ Cipher, ◆ Shadow, ✚ Patch, ⚡ Vex. Information is never conveyed by color alone.
  • Focus indicators: A bright cyan 3 px solid outline (#00ff9f) with glow shadow is shown on all focusable elements when navigated by keyboard. The outline contrasts at greater than 3:1 against dark backgrounds, meeting low-vision requirements. Mouse clicks suppress the ring.
  • All text colors meet WCAG AA 4.5:1 contrast ratio against their backgrounds in the default and high-contrast themes.

Low Vision Implemented

  • Relative font sizing: The base font is set to 1rem (16px) using relative units throughout. This means the entire UI scales with the user's browser default font size and with browser zoom.
  • Minimum text size: No UI element uses text smaller than 14px (0.875rem). All labels, badges, metadata, and chrome meet this minimum.
  • Line height: Body line height is 1.6 — exceeds the WCAG 1.4.12 minimum of 1.5. Narrative and agent card text use 1.7–2.1 line height.
  • Letter spacing: Body text uses 0.05em letter spacing to improve character legibility for users with low vision or reading difficulties.
  • Paragraph spacing: Paragraph margin is 1.5em — at least 1.5× the font size.
  • Zoom compatibility: At 200% browser zoom the layout adapts — the reasoning panel is hidden at viewports narrower than 960px; both sidebars are hidden below 640px, leaving a full-width single-column layout with no horizontal scrolling.
  • No text as images: All text — agent names, phase labels, status indicators, health percentages, dice results, and all badges — is real HTML text. Everything is selectable, searchable, and scales with zoom. The architecture diagram is a separate file used in documentation, not displayed in the game UI.
  • Health bars: Each health bar displays a numeric percentage text alongside the colour fill (e.g., "75%"). Information is never conveyed by colour alone.
  • OS magnification: No important content is placed exclusively in screen corners. All interactive controls are in the central flow of the header and main area.
  • Focus outline: The keyboard focus ring is 3px solid cyan (#00ff9f), meeting the WCAG 2.4.11 minimum and contrasting at greater than 3:1 against the dark background.

USING BROWSER ZOOM EFFECTIVELY

  • Press Ctrl++ / Cmd++ (Windows / macOS) or use Ctrl+scroll to zoom.
  • The game layout reflows automatically at high zoom — no horizontal scrolling required at any level.
  • To increase text size while keeping the 3-column layout, use the A / A+ / A++ button in the header instead of browser zoom. This scales narrative and agent card text (up to 22px) without shrinking the viewport.
  • Simple mode (≡ button) provides a full-width single-column layout at any font size.

RECOMMENDED BROWSER SETTINGS

  • Set your browser minimum font size to 14px in Settings → Appearance for additional protection against small text on any site.
  • Windows Magnifier (Win+= to activate) works with Ghost Protocol — no content is hidden at screen edges.
  • macOS Zoom (System Settings → Accessibility → Zoom) is supported — hover text zoom and full-screen zoom both function correctly.
  • If using ZoomText or similar overlay software, Ghost Protocol's layout does not use fixed-position elements that would be obscured by the overlay toolbar.

Cognitive Accessibility Implemented

  • /help command: Typing /help displays all available commands in plain language without a server round-trip.
  • Simple mode: The ≡ button in the header hides both sidebars, increases text size to 18 px base, removes all animations, and gives the narrative full width. Designed for players who prefer a distraction-free single-column layout.
  • Tooltips: Every panel in the reasoning sidebar and the sidebar sections display a plain-language tooltip on hover or keyboard focus explaining what the panel shows.
  • Who is speaking is always identified — agent name and role appear on every card before the response text.
  • The Vex moral-choice modal includes an optional third path ("Ask Patch") that reveals an ethical assessment before committing to an irreversible decision.

Mental Health Considerate Design Implemented

Ghost Protocol is built with mental health awareness as a non-negotiable design constraint, woven into the AI agents' system-level instructions rather than bolted on as an afterthought.

  • No gratuitous violence: Consequences are atmospheric and narrative — combat outcomes are described obliquely, never graphically. Violence is purposeful and consequence-aware; harm to civilians is always treated with moral weight.
  • Fundamentally hopeful tone: Even in the darkest heist moments, Ghost maintains that the crew has a way through. No dead ends, no despair spirals.
  • Morally difficult moments handled with care: When the narrative reaches an ethical crossroads, Ghost names it explicitly and offers a clear path forward. The Vex encounter is the primary example — a visible moral choice with an optional "Ask Patch for ethical guidance" option before committing.
  • Mission framing as data liberation: The crew's goal is extracting corporate data, not harming people. Ghost reinforces this framing throughout to keep the narrative purposeful rather than nihilistic.
  • No real-world harmful content: Agents are instructed at the system level to never generate information that could cause real-world harm, regardless of fictional framing. The game stays in Toronto 2047.
  • Pause and quit controls: Players can pause the game at any time with the Pause button or quit and return to the main screen without consequence. There is no penalty or pressure to continue.

Deaf and Hard of Hearing Implemented

Ghost Protocol is 100% text-based — a core design strength, not an accommodation. No audio is required at any point during gameplay.

  • No audio elements: There are no sound effects, music, audio notifications, or speech synthesis in Ghost Protocol. The codebase contains zero <audio> tags or Web Audio API calls. This was verified by a full source audit.
  • Alert state changes shown as text: Heat level transitions (COLD → WARM → HOT → SCORCHED) display as a text badge in the header (e.g. ALERT: HOT) and are announced via aria-live="assertive". Color is always supplementary — the label text names the state explicitly.
  • Dice roll results shown as text: Every dice roll displays the equation, total, and outcome label (e.g. d20 → 17 — SUCCESS) as visible text in the narrative stream and the reasoning panel. The full result is also logged to the mission log.
  • Agent responses identified by name: Every agent card shows the agent's name and role as the first visible element before their response text — no audio or voice differentiation required.
  • Vex encounter shown as a text dialog: The Vex encounter appears as a full-screen modal overlay with a text header, text body, and three clearly labelled choice buttons. No audio cue precedes or accompanies it.
  • Visual loading state on submission: When an action is submitted, the Send button is visibly disabled and a [ GHOST PROCESSING… ] indicator appears in the narrative stream. The button uses aria-busy="true" during processing.
  • Errors shown as visible text: All error states — server errors, blocked inputs, connection failures — render as visible text messages in the narrative stream. No audio accompanies errors.
  • Mission log is text only: Every game event (dice rolls, alert escalations, phase advances, agent actions) is logged as a timestamped text entry in the scrollable mission log.

Because Ghost Protocol is text-based by design, deaf and hard-of-hearing players have full access to every gameplay element without any setting changes or accommodations.

Mobility and Motor Accessibility Implemented

  • Full keyboard navigation: Every feature in Ghost Protocol is reachable by keyboard alone. Tab moves between interactive elements in logical order. No mouse or pointer device is required at any point.
  • 44×44 px minimum click targets: All buttons — Send, New Session, accessibility toggles, keyboard shortcut panel, Vex encounter choices, content advisory buttons, pause/resume, and overlay close — meet the WCAG 2.5.5 Enhanced target size of 44×44 CSS pixels.
  • No time limits: Ghost Protocol never times out waiting for player input. The Vex encounter modal waits indefinitely — no countdown, no auto-dismiss. Sessions never expire due to inactivity. Player activity is recorded (for transparency) but never used to force actions or end sessions.
  • No complex gestures: No swipe, drag-and-drop, multi-finger, or simultaneous multi-pointer gestures are required for any feature. Every action is a single click or tap.
  • No simultaneous keypresses: The only keyboard shortcut beyond single keys is Ctrl+/ (open shortcuts panel). This works with Windows Sticky Keys — press Ctrl first (it sticks), then press / — the browser fires the event with ctrlKey: true as expected. All other shortcuts (Enter, Escape, Arrow keys) are single keys.
  • Voice control compatibility: All interactive elements have visible text labels that voice control software can target:
    • Input field shows label "Enter your action" — say "click Enter your action"
    • Submit button shows "SEND" — say "click Send"
    • Session reset shows "NEW SESSION" — say "click New Session"
    • Accessibility toggles have descriptive aria-label attributes for Dragon NaturallySpeaking and similar tools
    No two interactive elements on the same screen share an identical label.
  • Pointer cancellation: Click actions trigger on mouse-up, not mouse-down. No custom mousedown handlers override default browser behavior — users can cancel an accidental click by moving the pointer away before releasing.
  • AAC and paste input: The action input field is a standard <input type="text"> with no character restrictions. It accepts pasted text from AAC devices, clipboard managers, and text expansion tools without modification.
  • Switch access: Tab + Enter navigation covers all game functionality. Players using single-switch or two-switch access can navigate to any element via Tab and activate it with Enter or Space.

See the Keyboard Shortcuts section below for the full list of shortcuts, and use Ctrl+/ in-game to open the shortcuts reference panel.

Semantic HTML Implemented

  • Proper landmark regions: <header>, <main>, <aside>, <section>, <footer>, <nav>.
  • Heading hierarchy: <h1> for the game title, <h2> for all sidebar and reasoning panel sections.
  • The player input field has an associated <label> (visually hidden) and an aria-describedby hint explaining how to use it.
  • Vex encounter choices are grouped with role="group" and aria-labelledby.
  • Objective items use <ul> with role="listitem" and per-item aria-label including status.

Known Limitations

  • No mobile / touch optimisation. While the layout now reflows at high zoom levels, it is designed for desktop screens and pointer input. Tap targets are not sized for mobile interaction.
  • ASCII banner. The decorative ASCII art logo is marked aria-hidden="true" with a text alternative (aria-label="Ghost Protocol ASCII art logo"). Screen readers will read the label only, not the art itself.
  • AI-generated content. The game's narrative is generated live by Azure OpenAI GPT-4o. Complex sentence structures and vocabulary may vary. We do not post-process for reading level.
  • Colour contrast in animated states. The blinking hot and scorched alert badges pulse between full and 60% opacity. Reduce motion disables this animation; the static colour meets AA contrast.

Screen Readers Supported

  • NVDA (Windows) with Firefox — primary development target
  • VoiceOver (macOS) with Safari — tested for macOS compatibility
  • JAWS (Windows) with Chrome — tested for enterprise screen reader compatibility
  • Narrator (Windows) — basic compatibility; some aria-live announcements may be delayed

WCAG 2.1 Compliance Target

Ghost Protocol targets WCAG 2.1 Level AA. The following criteria are addressed:

  • 1.1.1 Non-text Content — decorative elements are hidden from AT; meaningful icons have text alternatives.
  • 1.3.1 Info and Relationships — semantic HTML conveys structure.
  • 1.3.3 Sensory Characteristics — agent identification uses icons + color, not color alone.
  • 1.4.1 Use of Color — icons accompany all color-coded agent indicators.
  • 1.4.2 Audio Control — no audio content is present at any point; requirement is met trivially by design.
  • 1.4.3 Contrast (Minimum) — all text meets 4.5:1 against background.
  • 1.4.4 Resize Text — all text uses relative units (rem); page reflows at 200% zoom with no horizontal scrolling.
  • 1.4.10 Reflow — single-column layout activates automatically at narrow viewports (≤640px CSS pixels) and at high browser zoom levels.
  • 1.4.11 Non-text Contrast — focus rings and health bars meet 3:1.
  • 1.4.12 Text Spacing — line height ≥1.6, letter spacing 0.05em, paragraph spacing ≥1.5em. No content is clipped when these properties are overridden.
  • 2.1.1 Keyboard — all functionality available by keyboard alone; no mouse required.
  • 2.1.2 No Keyboard Trap — focus traps in modals are intentional and escapable.
  • 2.2.1 Timing Adjustable — no time limits are present; requirement met trivially by design. Sessions never expire due to inactivity.
  • 2.5.3 Label in Name — visible button text is contained in the accessible name of each control (e.g., visible "SEND" is in accessible name "Send action to the crew").
  • 2.5.5 Target Size (Enhanced) — all interactive controls are at least 44×44 CSS pixels (AAA criterion, implemented as a design standard).
  • 2.4.1 Bypass Blocks — skip navigation link provided.
  • 2.4.3 Focus Order — tab order follows reading order.
  • 2.4.7 Focus Visible — bright cyan focus ring on all interactive elements.
  • 3.1.1 Language of Page — lang="en" on html element.
  • 3.2.2 On Input — no unexpected context changes on input.
  • 3.3.1 Error Identification — blocked inputs produce descriptive error messages.
  • 4.1.2 Name, Role, Value — all interactive elements have accessible names, roles, and state.
  • 4.1.3 Status Messages — aria-live regions announce status without focus.

Report an Accessibility Issue

If you encounter a barrier while using Ghost Protocol, please report it. We aim to respond within 48 hours and resolve accessibility issues as a priority.

  • GitHub Issues: Open an issue tagged accessibility in the project repository.
  • Email: prashantibhatt04@gmail.com

Please include: the browser and screen reader you are using, the feature or page affected, and a description of what happened versus what you expected.

AI Transparency

Ghost Protocol's crew agents — Ghost (Game Master), Wraith, Cipher, Shadow, Patch, and Vex — are powered by Azure OpenAI (GPT-4o) via Microsoft Azure AI Foundry. All in-game dialogue, narrative synthesis, and tactical advice are AI-generated responses and should be treated as fiction.

Foundry IQ (the knowledge-base retrieval system) grounds agent responses in a synthetic lore corpus created specifically for this game. The eight knowledge documents describe the fictional Toronto 2047 setting, Nexus Corporation, and GenVault facility. No real-world personal data, proprietary information, or factual claims are stored in or retrieved from the knowledge base.

Dice rolls use Python's random.randint() (Mersenne Twister PRNG). No Azure API call is made for dice outcomes. The distribution is statistically uniform across d20 values — verified by 10,000-roll fairness tests in the project's test suite (tests/test_fairness.py).

All characters, corporations, locations, and events in Ghost Protocol are entirely fictional. The game is a creative writing and AI demonstration project built for the Microsoft Agents League Hackathon. No real hacking, exploitation, or harmful activities are facilitated or endorsed.
